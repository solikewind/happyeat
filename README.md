## 菜鸟杂谈

    如果有足够需求，你懂的。可以加入服务内部间的rpc接口，适合团队开发。可以做微服务，也可做单体模块化架构（加入gateway的概念）。

    个人我觉得目前即可，api调用也没问题，访问需求高的情况可以再考虑rpc。

| 调用链变长 | 原来一次 DB 查询可能变成：API → RPC → 另一个 RPC → DB |
| ---------- | ----------------------------------------------------- |

| 部署复杂度 | 从单进程变为多进程/多容器，K8s/编排、配置管理更复杂 |
| ---------- | --------------------------------------------------- |

| 开发与运维成本 | 需维护多套服务、部署、监控、链路追踪，本地联调更复杂 |
| -------------- | ---------------------------------------------------- |
